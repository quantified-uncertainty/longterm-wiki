## Purpose

These reports capture:

1. **Literature reviews** - Best practices from academic and industry sources
2. **Design decisions** - Why we built things a certain way
3. **Tool evaluations** - Comparisons of libraries and approaches
4. **Architecture notes** - How systems fit together

## Creating New Reports

When documenting significant technical decisions or research:

1. Create a new `.mdx` file in this directory
2. Include sources and references
3. Link to relevant code or components
4. Update the table above
